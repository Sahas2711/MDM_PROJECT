## Route 7: Voice Assistant (/voice-assistant)

```
VoiceAssistant.jsx

              └── POST /generate                          [voice-to-voice/app/main.py]

                    └── Returns: { audio_base64, audio_mime_type, provider, model, warnings }
  └── handleVoiceChat()
        └── fetchVoiceChat(audioFile)                     [api.js]
              └── POST /voice-chat                        [voice-to-voice/app/main.py]
                    └── orchestrator.voice_chat()
                          ├── Groq Whisper STT
                          ├── OpenRouter LLM
                          └── HuggingFace TTS
                    └── Returns: { transcription, response_text, response_audio_base64,
                                   stt_provider, llm_provider, tts_provider, warnings }
  └── Service health check on mount
        └── fetchVoiceHealth()
              └── GET /health                             [voice-to-voice/app/main.py]
```

**Integration Status**: FULLY INTEGRATED (when API keys are configured)
- All three voice pipeline stages are wired end-to-end
- WebSocket endpoint (/ws/voice-chat) exists in backend but NOT used by frontend (uses HTTP POST instead)
- Service will show "Unavailable" if voice service is not running or API keys are missing
