## Flow 1: Price Prediction (/predict)

```
User Input (Frontend)
  └── SellTiming.jsx: min_price (number), max_price (number)
        │
        ▼
API Call
  └── POST http://localhost:8000/predict?model_type=best
      Body: { "min_price": 1200.0, "max_price": 1800.0 }
        │
        ▼
FastAPI Route (main.py)
  └── predict() → predictor.predict(min_price, max_price, model_type="best")
        │
        ▼
Model Resolution (predict.py)
  └── _resolve("best") → settings.default_model → "random_forest"
  └── entry = _registry["random_forest"]  (ModelEntry with crop_price_classifier.pkl)
        │
        ▼
Feature Engineering (predict.py: build_feature_context)
  └── _resolve_commodity(crop_hint="") → "Apple" (default)
  └── _safe_encode("state", None, "Maharashtra") → state_encoded
  └── _safe_encode("district", None, "Pune") → district_encoded
  └── _safe_encode("market", None, "Pune APMC") → market_encoded
  └── _safe_encode("variety", "Apple", "Apple") → variety_encoded
  └── _safe_encode("grade", None, "FAQ") → grade_encoded
  └── _safe_encode("arrival_date", None, None) → arrival_date_encoded
  └── Returns dict with 9 encoded features + human-readable names
        │
        ▼
Feature Array Construction (predict.py: _feature_array)
  └── [feature_context[col] for col in _feature_columns]  ← order from feature_columns.pkl
  └── np.array([feature_values], dtype=np.float32)
        │
        ▼
Scaling (predict.py)
  └── scaled = _scaler.transform(features).astype(np.float32)
      (StandardScaler from feature_scaler.pkl; .astype(float32) fixes KMeans dtype issue)
        │
        ▼
Model Inference (predict.py: _predict_probabilities)
  └── entry.kind == "sklearn":
        probs = entry.model.predict_proba(scaled)[0]
  └── prediction_code = int(np.argmax(probs))
  └── prediction_label = MODEL_LABELS[prediction_code]  → "Low" / "Medium" / "High"
        │
        ▼
Intelligence Enrichment (intelligence.py: enrich)
  └── get_confidence() → predict_proba[prediction_code] (RF has predict_proba)
  └── get_price_range_analysis() → spread tier message
  └── get_market_insight() → (prediction_label, spread_tier) → actionable sentence
        │
        ▼
Recommendation Logic (predict.py: _recommendation_from_label)
  └── "High" → "SELL"
  └── "Medium" + confidence >= 0.75 → "SELL"
  └── else → "HOLD"
        │
        ▼
Clustering (predict.py: cluster_features)
  └── _kmeans.predict(scaled)[0] → cluster_id
  └── _kmeans.transform(scaled)[0][cluster_id] → cluster_distance
        │
        ▼
Feature Contributions (predict.py: _feature_contributions)
  └── Heuristic impact scores based on probabilities and price spread
  └── Sorted by impact descending
        │
        ▼
Response Assembly
  └── Returns full dict: model_used, prediction_label, recommendation, confidence,
      probabilities, price_range_analysis, market_insight, cluster_id, cluster_distance,
      feature_context, feature_contributions, latency_ms
        │
        ▼
Frontend Rendering (SellTiming.jsx)
  └── Recommendation badge (SELL/HOLD)
  └── Confidence percentage
  └── Market insight text
  └── Price range analysis text
  └── Model name + version + latency
```

## Flow 2: Image Freshness Check (/predict-image)

```
User Input (Frontend)
  └── Dashboard.jsx or ModelPerformance.jsx: imageFile (File object)
        │
        ▼
API Call
  └── POST http://localhost:8000/predict-image
      Body: multipart/form-data { file: imageFile }
        │
        ▼
FastAPI Route (main.py)
  └── File validation: content_type in ALLOWED_CONTENT_TYPES, size <= 10MB, not empty
  └── image_predictor.predict_image(image_bytes)
        │
        ▼
Image Decoding (predict_image.py: _decode_image)
  └── cv2.imdecode(np.frombuffer(image_bytes), cv2.IMREAD_COLOR)
  └── cv2.cvtColor(bgr, cv2.COLOR_BGR2RGB)
  └── Image.fromarray(rgb)  → PIL Image
  └── Fallback: PIL Image.open(BytesIO(image_bytes)).convert("RGB")
        │
        ▼
Fruit Domain Check (predict_image.py: _predict_fruit_domain)
  └── YOLOv8 detection:
        rgb = np.array(image)
        bgr = cv2.cvtColor(rgb, cv2.COLOR_RGB2BGR)
        detections = _yolo_model(bgr, verbose=False)
  └── For each detection:
        if detector_label not in YOLO_ALLOWED_LABELS: skip
        crop = rgb[y1:y2, x1:x2]  ← bounding box crop
        preds = _fruit_model.predict(_preprocess_cv(crop, (224,224)))
        if confidence > best_conf: update best
  └── fruit_detected = best_detector_conf >= 0.35
        │
        ▼ (if fruit_detected == False → return early with "Not Fruit")
        │
        ▼
Freshness Inference (predict_image.py: predict_image)
  └── _preprocess_pil(image, (128,128)):
        image.resize((128,128), Image.BILINEAR)
        np.array(resized) / 255.0
        np.expand_dims(arr, axis=0)
  └── probs = _freshness_model.predict(preprocessed, verbose=0)[0]
  └── class_idx = np.argmax(probs)
  └── freshness = FRESHNESS_CLASS_NAMES[class_idx]  → "Fresh" or "Rotten"
        │
        ▼
Response
  └── { freshness, confidence, model_version, latency_ms, image_reason,
         fruit_detected, fruit_confidence, fruit_label, fruit_reason,
         preprocessing: {original_size, resized_to, normalized, decoder} }
```

## Flow 3: Smart Decision Pipeline (/smart-decision)

```
User Input (Frontend)
  └── SmartDecision.jsx: imageFile (File), cropHint (string, optional)
        │
        ▼
API Call
  └── POST http://localhost:8000/smart-decision?crop_hint=apple&model_type=best
      Body: multipart/form-data { file: imageFile }
        │
        ▼
FastAPI Route (main.py)
  └── File validation (same as /predict-image)
  └── await decision_engine.decide(image_bytes, crop_hint, model_type)
        │
        ▼
Node 1: PREPROCESSING (smart_decision.py → predict_image.py)
  └── image_result = image_predictor.analyze_image(image_bytes)
      (same as /predict-image flow above)
        │
        ▼ (if fruit_detected == False → return early response with all fields null/skipped)
        │
        ▼
Node 2: IMAGE_VALIDATION
  └── fruit_detected = image_result["fruit_detected"]
  └── fruit_confidence = image_result["fruit_confidence"]
        │
        ▼
Node 3: QUALITY_ANALYSIS
  └── freshness = image_result["freshness"]  → "Fresh" or "Rotten"
  └── cnn_confidence = image_result["confidence"]
        │
        ▼
Node 4: MARKET_ENRICHMENT (market_data.py: fetch_market_prices)
  └── keyword = _keyword_from_hint(crop_hint)
  └── query = f"{crop_hint} mandi price India today per quintal"
  └── DuckDuckGo API call (4s timeout):
        GET https://api.duckduckgo.com/?q=...&format=json
        Parse Abstract, Answer, RelatedTopics for price numbers
  └── If no prices found → fallback to _FALLBACK_PRICES[keyword]
  └── Returns: { min_price, max_price, source, query, matched_crop, source_reason }
        │
        ▼
Node 5: FEATURE_PIPELINE + PRICE_MODEL (predict.py: predict)
  └── price_result = price_predictor.predict(
          price_data["min_price"], price_data["max_price"],
          model_type=model_type, crop_hint=crop_hint
      )
  └── (Full price prediction flow as in Flow 1)
        │
        ▼
Node 6: CLUSTER_ANALYSIS
  └── cluster_id, cluster_distance from price_result
        │
        ▼
Node 7: DECISION_ENGINE (smart_decision.py)
  └── if freshness == "Rotten" → final_recommendation = "DO NOT SELL"
  └── else → final_recommendation = price_result["recommendation"]
  └── quality_adj = 0.95 if Fresh else 0.60
  └── final_price = mid_price * quality_adj
        │
        ▼
Node 8: HOLD_STRATEGY (_build_hold_plan)
  └── if freshness == "Rotten" → should_hold=False, hold_days=0
  └── if recommendation == "SELL" → should_hold=False, hold_days=settings.hold_days_high_price (1)
  └── if "Medium" → should_hold=True, hold_days=settings.hold_days_medium_price (4)
  └── if "Low" → should_hold=True, hold_days=settings.hold_days_low_price (7)
        │
        ▼
Node 9: NARRATIVE GENERATION (generative.py: generate_analysis)
  └── Concatenates: fruit_reason + image_reason + price_source_reason +
                    price_reason + decision_reason + hold_reason + hold_instructions
        │
        ▼
Node Graph Construction (_build_node_graph)
  └── 9 nodes with id, name, node, status, depends_on, data, input, output, error, why
  └── 8 linear edges
  └── Node statuses: SUCCESS / FAILED / PARTIAL / SKIPPED
        │
        ▼
Response Assembly
  └── Full SmartDecisionResponse: ~35 fields including pipeline_result, node_graph, workflow
        │
        ▼
Frontend Rendering (SmartDecision.jsx)
  └── Image with overlay (freshness, fruit confidence, decision)
  └── InfoCards: recommendation, freshness confidence, hold guidance
  └── Price probabilities, cluster signal, market price
  └── Feature contributions bar chart
  └── Model context (state, district, market, commodity)
  └── Full analysis text
  └── WorkflowNode graph (9 collapsible nodes with input/output/why)
```

## Flow 4: Voice Pipeline (/voice-chat)

```
User Input (Frontend)
  └── VoiceAssistant.jsx: audioFile (WAV/MP3/WEBM/OGG/MP4/M4A)
        │
        ▼
API Call
  └── POST http://localhost:8001/voice-chat
      Body: multipart/form-data { file: audioFile }
        │
        ▼
FastAPI Route (voice-to-voice/app/main.py)
  └── validate_audio_upload(file, audio_bytes)
  └── await orchestrator.voice_chat(audio_bytes, filename, content_type)
        │
        ▼
STT Stage (orchestrator.py → providers/stt.py)
  └── Groq API: POST https://api.groq.com/openai/v1/audio/transcriptions
      Model: whisper-large-v3
  └── Returns: transcription text (Marathi/Hindi/mixed)
        │
        ▼
LLM Stage (orchestrator.py → providers/llm.py)
  └── OpenRouter API: POST https://openrouter.ai/api/v1/chat/completions
      Model: meta-llama/llama-3.3-70b-instruct:free
      System prompt: Marathi-first farmer support (from prompts.py)
  └── Returns: Marathi response text
        │
        ▼
TTS Stage (orchestrator.py → providers/tts.py)
  └── HuggingFace Inference API: POST https://api-inference.huggingface.co/models/facebook/mms-tts-mar
  └── Returns: audio bytes (WAV)
        │
        ▼
Response Assembly
  └── { transcription, response_text, response_audio_base64, response_audio_mime_type,
         stt_provider, llm_provider, llm_model, tts_provider, warnings }
        │
        ▼
Frontend Rendering (VoiceAssistant.jsx)
  └── Transcription text display
  └── Marathi response text display
  └── Audio player (base64 → data URL → <audio> element)
  └── Provider pills (STT / LLM / TTS)
```

## Training Pipeline (Notebook → Production)

```
dataset/crop_prices.csv
  └── pandas read_csv
  └── Feature engineering: LabelEncoder for categorical columns
  └── StandardScaler fit on 9 features
  └── Train/test split
        │
        ├── Random Forest training → crop_price_classifier.pkl
        ├── ANN training (Keras) → ann_model.h5
        ├── DNN training (Keras) → dnn_model.h5
        ├── VAE training (Keras) → vae_weights.h5 [NOT DEPLOYED]
        ├── KMeans training → kmeans_clusterer.pkl
        └── Export: feature_scaler.pkl, feature_columns.pkl, label_encoders.pkl
              │
              ▼
        backend/models/ (manual copy from Colab download)
              │
              ▼
        predict.py loads all at startup via lifespan()
              │
              ▼
        Inference on every /predict and /smart-decision request
```
