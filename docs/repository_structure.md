## Hidden Architecture Patterns

1. **Dual scaler/model system**: Two parallel model stacks exist — legacy 2-feature (crop_model.pkl + scaler.pkl + features.pkl) and current 9-feature (crop_price_classifier.pkl + feature_scaler.pkl + feature_columns.pkl). The backend has a fallback: if crop_price_classifier.pkl is missing, it loads crop_model.pkl. The streamlit frontend uses only the legacy stack.

2. **Three separate servers**: The project runs 3 independent processes — FastAPI backend (port 8000), React/Vite frontend (port 5173), and voice-to-voice FastAPI microservice (port 8001). No service mesh or API gateway.

3. **YOLO → fruit_classifier pipeline**: predict_image.py runs YOLOv8 first to detect and crop the fruit bounding box, then passes the crop to fruit_classifier.h5 (224x224). The freshness CNN (128x128) runs on the full image independently. This is a two-stage vision pipeline.

4. **Node graph as API response**: smart_decision.py builds a complete n8n-style workflow node graph (9 nodes, 8 edges, with status/input/output/why per node) and returns it in the API response. The frontend renders this as an interactive collapsible graph.
