## Compatibility Risks

### HIGH Risk
1. **TensorFlow 2.16 + Keras 3 + .h5 files**: TF 2.16 ships with Keras 3 by default. Loading `.h5` files saved with Keras 2 (from Colab) may require `tf_keras` package or `compile=False` flag. The code uses `compile=False` which mitigates this, but model architecture compatibility is not guaranteed.

2. **KMeans float32 dtype issue**: The code explicitly documents this: `StandardScaler.transform()` outputs float64; KMeans C extension requires float32. Fix is applied (`scaled.astype(np.float32)`), but this is a known fragile point.

3. **YOLOv8 + PIL patch conflict**: The code uses cv2 as primary decoder specifically to avoid Ultralytics PIL patch issues. This is a documented workaround, not a clean fix.

### MEDIUM Risk
4. **Vite 8 + React 19**: Both are very recent releases. Plugin ecosystem may have compatibility gaps.

5. **DuckDuckGo API**: Not an official API. No SLA, no authentication, subject to change or blocking. The fallback hardcoded prices are always used in practice.

6. **CORS allow_origins=["*"]**: Both FastAPI servers have wildcard CORS. Acceptable for development, security risk in production.

7. **No authentication on any endpoint**: All prediction endpoints are publicly accessible.

### LOW Risk
8. **framer-motion + recharts bundle size**: README acknowledges chunk size warnings. No route-level code splitting implemented.

9. **React 19 ecosystem lag**: Some third-party libraries may not yet support React 19 concurrent features.
